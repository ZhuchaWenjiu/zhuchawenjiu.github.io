C++ 异常处理是一种用于处理程序运行时错误的机制。当在程序执行期间发生错误时，C++ 可以通过抛出和捕获异常来实现对这些错误的处理。异常处理允许程序员将错误处理代码与正常程序逻辑分开，提高了程序的可维护性和可读性。

  1. 异常的基本语法：

    • 抛出异常： 使用 throw 关键字抛出异常。

    • 捕获异常： 使用 try、catch 和 throw 关键字捕获异常。

  2. 异常类型：

    • 异常可以是任何类型的对象，包括内置类型、类类型或指针类型。

    • 标准库中有一些预定义的异常类型，如 std::exception，派生类包括 std::runtime_error、std::logic_error 等。

  3. 捕获多个异常：

    • 可以使用多个 catch 块来捕获不同类型的异常。

  4. 异常对象的传递：

    • 异常对象可以传递到调用栈的更高层，直到找到匹配的 catch 块。

  5. RAII 与异常安全：

    • RAII（Resource Acquisition Is Initialization）技术有助于确保在异常发生时资源能够被正确释放。

    • 使用智能指针、标准库容器等可以提高程序的异常安全性。

  6. 异常规范（Exception Specification）：

    • 异常规范是一种旧的 C++ 特性，允许在函数声明中指定函数可能抛出的异常类型。

    • 在现代 C++ 中，更推荐使用 noexcept 关键字。

  7. std::exception 和派生类：

    • std::exception 是标准库中定义的基本异常类，其他异常类通常从它派生。

    • std::runtime_error 和 std::logic_error 是两个常用的派生类。

  8. 自定义异常类：

    • 可以根据需要创建自定义的异常类，继承自 std::exception 或其派生类。

    • 自定义异常类通常应该提供适当的构造函数，以便传递错误信息。
