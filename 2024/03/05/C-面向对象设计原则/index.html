面向对象设计中有一些基本的设计原则，其中 SOLID 是一个常见的缩写，代表了五个重要的设计原则。以下是这些原则的简要介绍：

  1. 单一职责原则（Single Responsibility Principle - SRP）：

    • 一个类应该只有一个修改的理由。换句话说，一个类应该只有一个职责。

  2. 开放/封闭原则（Open/Closed Principle - OCP）：

    • 软件实体（类、模块、函数等）应该对扩展开放，对修改封闭。这意味着可以通过添加新的代码来扩展功能，而不是修改现有代码。

  3. 里氏替换原则（Liskov Substitution Principle - LSP）：

    • 子类型必须能够替换它们的基类型，而不影响程序的正确性。在使用继承时，子类应该继承父类的行为，但可以替换掉需要修改的部分。

  4. 接口隔离原则（Interface Segregation Principle - ISP）：

    • 客户端不应该被迫依赖于它们不使用的接口。一个类不应该被强迫实现它用不到的接口。

  5. 依赖反转原则（Dependency Inversion Principle - DIP）：

    • 高层模块不应该依赖于低层模块，两者都应该依赖于抽象。抽象不应该依赖于具体实现，具体实现应该依赖于抽象。

这些原则有助于编写灵活、可维护且易于理解的面向对象代码。通过遵循这些原则，可以减少代码的复杂性，提高代码的可复用性和可扩展性。

继承是面向对象编程（OOP）中的一种重要概念，允许一个类（称为子类或派生类）继承另一个类（称为父类或基类）的属性和方法。这使得代码重用、抽象和多态成为可能。以下是C++中继承的详细解释：

面向对象的特点

  • 封装：权限访问控制符

  • 继承：基类和派生类

  • 多态：虚函数和抽象类

2. 派生类的构造和析构：

  • 构造函数和析构函数的调用顺序：

    • 在派生类的对象被创建时，首先调用基类的构造函数，然后再调用派生类的构造函数。

    • 在派生类的对象被销毁时，先调用派生类的析构函数，然后再调用基类的析构函数。

    创建从内至外，销毁从外至内

    {[( obj )]} 先从()到 [] 再到 {} ，销毁从 {} 到 [] 再到 ()

3. 访问权限：

父类的 private 成员对于子类都是不可访问的！！！！

父类的 public 和 protected 成员会根据继承权限变成子类的不同权限成员

友元函数定义在哪里，它就可以访问定义位置对象的所有成员

public : 可以被类外访问

protected ： 不可以被类外访问，如果不继承时，跟private 权限一样，为了解决 private 继承后子类不可访问出现的

private: 不可被类外访问，继承后对子类不可见

  • 公有继承（public inheritance）：

    • 在公有继承中，基类的公有成员在派生类中仍然是公有的，保护成员在派生类中仍然是保护的。

  • 私有继承（private inheritance）：

    • 在私有继承中，基类的所有成员在派生类中都变为私有的。

  • 保护继承（protected inheritance）：

    • 在保护继承中，基类的公有和保护成员在派生类中都变为保护的。

4. 多重继承：

C++ 允许一个类从多个类中继承成员。在多重继承中，一个类可以同时继承多个基类的属性和方法。

5. 虚函数和多态性：

  • 虚函数（virtual functions）：

    • 在基类中声明为虚函数的函数，可以在派生类中被重写（覆盖）。

  • 动态绑定和静态绑定

动态绑定的函数是在运行期间确认的，因为子类可以隐式转换为父类，因此对于下面这个容器，它可以存放不同类型的对象，这些对象在编译的时候都是按照基类看的，因此他们只会调用基类的PrintClassName 但是如果用了虚函数，就可以在调用时直接调用 overload 后的方法，这种叫动态绑定。

虚函数都是动态绑定，普通的都是静态

  • 多态性（polymorphism）：

    • 多态性允许使用基类的指针或引用来调用派生类的虚函数，以实现动态绑定。

6. 抽象类和纯虚函数：

C++中的抽象类没有关键字，声明纯虚函数后就是了

  • 抽象类（abstract class）：

    • 包含至少一个纯虚函数的类被称为抽象类，不能实例化对象。

  • 纯虚函数（pure virtual function）：

    • 纯虚函数是在基类中声明的虚函数，但没有提供实现。子类必须实现纯虚函数，否则它们也会成为抽象类。
